# HS Battlegrounds Sandbox

A Hearthstone Battlegrounds battle simulator. Build boards, simulate fights, watch replays.

## Project Structure

```
hs-battlegrounds/
├── backend/                 # FastAPI + SQLite
│   ├── app/
│   │   ├── api/             # Routes: /api/cards, /api/simulate
│   │   ├── core/            # Database (SQLite)
│   │   └── services/
│   │       ├── blizzard.py  # Card loader into DB
│   │       ├── cards_data.py# Static dataset (fallback)
│   │       ├── cards_scraped.json  # ← generated by fetch_cards.py
│   │       └── simulator.py # Battle simulation engine
│   ├── Dockerfile
│   ├── requirements.txt
│   └── startup.py           # DB initialization on startup
├── frontend/                # React (Vite) + nginx
│   ├── src/
│   ├── nginx-frontend.conf  # nginx: static files + proxy /api/ → backend
│   └── Dockerfile
├── scripts/
│   ├── fetch_cards.py       # ← CARD PARSER (run manually)
│   └── cards_manual.json    # Manual stat overrides
└── docker-compose.yml       # 2 containers: frontend + backend
```

## Quick Start

```bash
git clone <repo-url>
cd hs-battlegrounds
docker compose up --build
```

Open http://localhost

## Updating Cards (Latest Patch)

The `scripts/fetch_cards.py` script scrapes **bgcheatsheet.com** (current card pool)
and enriches the data with **HearthstoneJSON** (stats, tiers, card text) where available.

### Step 1: Install dependencies

```bash
pip install requests beautifulsoup4 lxml
```

### Step 2: Run the script

```bash
python scripts/fetch_cards.py
```

The script will:
1. Download all tribes from bgcheatsheet.com (Beast, Demon, Dragon, ...)
2. Attempt to load stats from HearthstoneJSON
3. Apply manual overrides from `scripts/cards_manual.json`
4. Save the result to `backend/app/services/cards_scraped.json`

### Step 3: Uncomment the volume in docker-compose.yml

```yaml
volumes:
  - db-data:/app/data
  - ./scripts/cards_scraped.json:/app/data/cards_scraped.json:ro  # ← uncomment this
```

### Step 4: Restart

```bash
docker compose down
docker compose up --build
```

The backend will automatically detect `cards_scraped.json` and load cards from it.

### Manual Stat Overrides

If HearthstoneJSON is unavailable or the data is inaccurate — edit `scripts/cards_manual.json`:

```json
{
  "Cave Hydra": {"tier": 3, "attack": 2, "health": 4, "mechanics": ["cleave"]},
  "Brann Bronzebeard": {"tier": 4, "attack": 2, "health": 4}
}
```

After editing, re-run `fetch_cards.py` and restart docker compose.

## API

| Method | URL | Description |
|--------|-----|-------------|
| GET | `/api/cards/` | Card list (filters: tier, race, mechanic) |
| GET | `/api/cards/{id}` | Single card |
| POST | `/api/simulate/battle` | Monte Carlo simulation (1000 iterations) |
| POST | `/api/simulate/replay` | Step-by-step battle replay |
| POST | `/api/cards/sync` | Force reload cards into DB |
| GET | `/api/health` | Healthcheck |
| GET | `/docs` | Swagger UI |

## Deploying to a Server

1. Get a VPS and point your domain's A record to the server IP
2. On the server:

```bash
# Install Docker
curl -fsSL https://get.docker.com | sh

# Clone the repo
git clone <repo-url>
cd hs-battlegrounds

# Start
docker compose up -d --build
```

3. For HTTPS add Certbot (Let's Encrypt):

```bash
apt install certbot python3-certbot-nginx
certbot --nginx -d yourdomain.com
```

## Tech Stack

- **Frontend**: React 18 + Vite, Zustand, @dnd-kit, Framer Motion
- **Backend**: Python 3.11, FastAPI, SQLite (aiosqlite)
- **Infra**: Docker Compose, nginx (2 containers)
